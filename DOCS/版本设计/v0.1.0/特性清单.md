# crystal-adk v0.1.0 特性清单

本文档只记录 v0.1.0 当前已落地的代码能力、对应文件与当前边界，不讨论设计推演过程。

## 1. 版本定位

当前版本已经具备：

- 面向 C# / .NET 的轻量 Agent Runtime
- 外部统一使用入口，内部按厂商拆分具体 Provider 实现
- 支持非流式对话 `RunAsync(...)`
- 支持普通文本流式输出 `StreamTextAsync(...)`
- 支持 Agent 事件流 `StreamEventsAsync(...)`
- 支持 Tool Calling 主循环
- 已优先适配 `Ollama` 与 `火山引擎方舟协议（ARK）`

当前版本还不包含：

- ASP.NET Core Web 宿主层
- DI 扩展
- 持久化会话存储
- Prompt 模板系统
- 多 Agent / Handoff
- ZAI Provider 实现代码

## 3. Core 层

### 3.1 `Core/AgentContracts.cs`

对应特性：

- 定义运行时消息模型 `RuntimeMessage`
- 定义工具调用模型 `RuntimeToolCall`
- 定义工具描述 `AgentToolDescriptor`
- 定义工具运行记录 `ToolRunRecord`
- 定义运行请求 `AgentRunRequest`
- 定义运行结果 `AgentRunResult`
- 定义工具接口 `IAgentTool`

对应能力：

- 统一四类消息角色：`system` / `user` / `assistant` / `tool`
- 统一 Assistant Tool Call 协议
- 统一 Tool Result 回填格式

### 3.2 `Core/AgentHostOptions.cs`

对应特性：

- 支持设置 `SystemPrompt`
- 支持设置 `MaxRounds`

对应能力：

- 控制默认系统提示词
- 控制 Tool Loop 最大轮次

### 3.3 `Core/ToolHub.cs`

对应特性：

- Tool 注册
- Tool 描述查询
- Tool 按名称调用

对应能力：

- `AgentHost` 不直接依赖业务工具实现
- 所有工具通过 `ToolHub` 统一注册和分发

当前边界：

- 只支持运行期内存注册
- 不支持自动程序集扫描
- 不支持插件式动态加载

### 3.4 `Core/Providers.cs`

对应特性：

- 定义 `IChatProvider`
- 定义 `StreamingChatChunk`

对应能力：

- 非流式完成接口 `CreateCompletionAsync(...)`
- 流式完成接口 `StreamCompletionAsync(...)`
- 统一 provider 输出文本增量、思考增量、工具快照、结束标记

### 3.5 `Core/Events.cs`

对应特性：

- 定义 `AgentEvent`
- 定义 `AgentEventKinds`

当前统一事件标识：

- `run_started`
- `tool_call_started`
- `tool_call_completed`
- `text_delta`
- `final_answer`
- `run_completed`
- `run_failed`

### 3.6 `Core/AgentHost.cs`

对应特性：

- 非流式主循环 `RunAsync(...)`
- 内部流式执行主线 `RunStreamingAsync(...)`
- `CreateSession()`
- 简化重载 `AgentHost(IChatProvider, AgentHostOptions?)`
- 完整重载 `AgentHost(ToolHub, IChatProvider, AgentHostOptions?)`

对应能力：

- 自动补充 system message
- 调用 provider 获取 assistant
- 自动识别 assistant 的 tool calls
- 自动调用 ToolHub
- 自动写回 tool message
- 多轮循环直到拿到最终回答
- 防止重复 Tool Call
- 记录 tool 耗时
- 在流式执行中同时维护运行状态与最终结果

当前边界：

- 无工具场景可直接使用简化重载
- 有工具场景仍需显式传入 `ToolHub`
- 当前不包含内建重试策略
- 当前不包含上下文裁剪策略

## 4. Session 层

### 4.1 `Session/AgentSession.cs`

对应特性：

- Session 内部维护 `_messages`
- 暴露只读 `Messages`
- 非流式发送 `RunAsync(...)`
- 文本流发送 `StreamTextAsync(...)`
- 事件流发送 `StreamEventsAsync(...)`
- 会话重置 `Reset()`

对应能力：

- 外部项目不需要手动维护消息数组主循环
- `Session` 负责在发送前追加 `user` message
- 运行完成后自动把最终 `messages` 写回会话

当前实现结论：

- `RunAsync(...)` 是非流式主接口
- `StreamTextAsync(...)` 是普通文本流式输出接口
- `StreamEventsAsync(...)` 是完整 Agent 事件流接口
- 流式接口当前不会在结束后再偷偷补发第二次 `RunAsync(...)`

### 4.2 `Session/TextChunk.cs`

对应特性：

- 定义文本流块对象 `TextChunk`

对应能力：

- `StreamTextAsync(...)` 对外只暴露文本增量，不暴露底层事件细节

## 7. 当前版本已确认的设计结论

- `AgentSession` 负责会话内消息状态
- 外部项目可以从 `Session.Messages` 取出完整消息数组
- `RunAsync(...)` 是非流式接口
- `StreamTextAsync(...)` 是普通文本流式输出接口
- `StreamEventsAsync(...)` 是 Agent 事件流接口
- 无工具场景不应强迫开发者手写 `ToolHub`
- `AgentHost` 已提供无工具简化重载
- Provider 外部体验统一，内部实现按厂商拆分
- Tool 协议差异由 Provider 适配层与 Runtime 内部消化
- Event 名称由 `crystal-adk` 定义，而不是直接暴露厂商原始事件
