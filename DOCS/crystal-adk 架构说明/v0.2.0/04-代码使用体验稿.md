# crystal-adk v0.2.0 代码使用体验稿

## 场景一：开发者手动初始化 provider options

```csharp
using Crystal.Adk.Providers;

// 包层不负责读取环境变量或配置文件
// 开发者自己决定 options 的来源
var options = new ChatProviderOptions
{
    Vendor = "ark",
    ApiKey = "your-api-key",
    Model = "doubao-seed-2-0-pro-260215",
    BaseUrl = "https://ark.cn-beijing.volces.com/api/v3/chat/completions",
    TimeoutMs = 60000,
    Temperature = 0.7,
    TopP = 0.9,
    MaxOutputTokens = 4096,
    EnableThinking = true
};
```

使用特征：

```text
配置初始化权归开发者
包层只定义数据结构
thinking 是否开启由开发者自己表达
```

## 场景二：开发者手动初始化历史消息

```csharp
using Crystal.Adk.Abstractions;
using Crystal.Adk.Session;

// 消息数组初始化权归开发者
// 可以单 system、多 system、无 system
var history = new SessionMessageManager(new[]
{
    new RuntimeMessage { Role = "system", Content = "静态提示词" },
    new RuntimeMessage { Role = "system", Content = "动态提示词" }
});
```

使用特征：

```text
不内置 SystemPrompt
不自动补 system
开发者决定初始上下文形状
```

## 场景三：开发者手动装配 provider 与 session

```csharp
using Crystal.Adk.Abstractions;
using Crystal.Adk.Providers;
using Crystal.Adk.Session;

using var httpClient = new HttpClient();

var options = new ChatProviderOptions
{
    Vendor = "ark",
    ApiKey = "your-api-key",
    Model = "doubao-seed-2-0-pro-260215",
    BaseUrl = "https://ark.cn-beijing.volces.com/api/v3/chat/completions",
    TimeoutMs = 60000,
    Temperature = 0.7,
    TopP = 0.9,
    MaxOutputTokens = 4096,
    EnableThinking = true
};

var history = new SessionMessageManager(new[]
{
    new RuntimeMessage { Role = "system", Content = "静态提示词" },
    new RuntimeMessage { Role = "system", Content = "动态提示词" }
});

var provider = ChatProviderFactory.Create(httpClient, options);
var session = new AgentSession(provider, history);
```

使用特征：

```text
session 是业务侧操作入口
provider 是底层协议执行对象
history 是独立消息管理对象
```

## 场景四：最小非流式对话

```csharp
// 业务侧只传本轮 userInput
var assistant = await session.RunAsync("你好");

Console.WriteLine(assistant.ThinkingContent);
Console.WriteLine(assistant.Content);
```

内部形态：

```text
session.AddUser
-> history.Export
-> provider.CompleteAsync
-> history.Add(assistant)
```

## 场景五：最小文本流对话

```csharp
await foreach (var chunk in session.StreamTextAsync("解释一下设备履历分析"))
{
    Console.Write(chunk.ThinkingText);
    Console.Write(chunk.Text);
}
```

内部形态：

```text
session.AddUser
-> history.Export
-> provider.StreamAsync
-> TextChunk(ThinkingText + Text)
-> history.AddAssistant(fullText, fullThinking)
```

## 场景六：直接编辑历史消息

```csharp
// 修改首条 system
history.UpdateFirstSystem("新的系统提示词");

// 修改最新 user
history.UpdateLatestUser("修正后的用户输入");

// 删除最新 assistant
history.RemoveLatestAssistant();
```

使用特征：

```text
消息历史编辑权在 SessionMessageManager
不把消息编辑逻辑塞进 AgentSession
```

## 场景七：导出消息用于调试或自定义发送

```csharp
var messages = history.Export();

foreach (var message in messages)
{
    Console.WriteLine($"{message.Role}: {message.Content}");
    Console.WriteLine($"{message.Role}.thinking: {message.ThinkingContent}");
}
```

使用特征：

```text
消息历史可导出
消息历史可复制
便于后续做上下文裁剪和重组
thinking 可随历史一起透传
```
