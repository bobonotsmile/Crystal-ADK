# crystal-adk v0.1.0 代码使用体验稿

## 典型伪应用场景

### 场景一：最小非流式对话应用

```csharp
using Crystal.Adk.Core;
using Crystal.Adk.Providers;

var httpClient = new HttpClient();

// 统一 provider 配置入口
var provider = ChatProvider.Create(httpClient, new ChatProviderSettings
{
    Vendor = "ark",
    ApiKey = "your-api-key",
    Model = "doubao-seed-2-0-pro-250415",
    BaseUrl = "https://ark.cn-beijing.volces.com/api/v3/chat/completions",
    TimeoutMs = 30000,
    EnableThinking = false
});

// Host 持有 provider、system prompt、max rounds
var host = new AgentHost(provider, new AgentHostOptions
{
    SystemPrompt = "你是一个简洁的助手。",
    MaxRounds = 8
});

// 业务侧真正长期持有的是 session
var session = host.CreateSession();

// 只传本轮 userInput，不手动传历史消息
var result = await session.RunAsync("你好");

// 结果里有最终文本，也有运行后的完整消息历史
Console.WriteLine(result.FinalMessage);
```

使用体验要点：

- provider 创建统一
- session 入口直观
- 历史消息自动维护
- Host 必须存在

### 场景二：最小文本流应用

```csharp
using Crystal.Adk.Core;
using Crystal.Adk.Providers;

var httpClient = new HttpClient();
var provider = ChatProvider.Create(httpClient, new ChatProviderSettings
{
    Vendor = "ollama",
    Model = "qwen3:8b",
    BaseUrl = "http://127.0.0.1:11434/api/chat",
    TimeoutMs = 30000
});

var host = new AgentHost(provider, new AgentHostOptions
{
    SystemPrompt = "你是一个简洁的助手。"
});

var session = host.CreateSession();

// 对外是 await foreach 的文本流形态
await foreach (var chunk in session.StreamTextAsync("解释一下设备履历分析"))
{
    Console.Write(chunk.Text);
}
```

使用体验要点：

- 流式消费形态自然
- session 会在流结束后自动补齐完整 assistant 历史
- provider 原始 SSE/NDJSON 被隐藏

### 场景三：统一事件流应用

```csharp
using Crystal.Adk.Core;
using Crystal.Adk.Providers;

var httpClient = new HttpClient();
var provider = ChatProvider.Create(httpClient, new ChatProviderSettings
{
    Vendor = "ark",
    ApiKey = "your-api-key",
    Model = "doubao-seed-2-0-pro-250415"
});

var host = new AgentHost(provider);
var session = host.CreateSession();

// 这里拿到的不是 provider 原始流，而是 Host 统一生命周期事件
await foreach (var evt in session.StreamEventsAsync("查询设备 EQ-001 并总结状态"))
{
    Console.WriteLine($"{evt.Kind} | {evt.ToolName} | {evt.Text}");
}
```

使用体验要点：

- 统一事件流便于观察运行过程
- 名字上容易让人误以为是 provider 原始流

### 场景四：带工具的伪业务应用

```csharp
using Crystal.Adk.Core;
using Crystal.Adk.Providers;

var httpClient = new HttpClient();
var provider = ChatProvider.Create(httpClient, new ChatProviderSettings
{
    Vendor = "ark",
    ApiKey = "your-api-key",
    Model = "doubao-seed-2-0-pro-250415"
});

var toolHub = new ToolHub();
toolHub.Register(new QueryDeviceInfoTool());
toolHub.Register(new QueryRepairHistoryTool());

var host = new AgentHost(toolHub, provider, new AgentHostOptions
{
    SystemPrompt = "你是设备运维助手。"
});

var session = host.CreateSession();

// 业务侧不手动执行工具；Host 自动做 tool loop
var result = await session.RunAsync("查询设备 EQ-001 的状态和最近维修记录");

Console.WriteLine(result.FinalMessage);
```

使用体验要点：

- 自动 tool loop 开箱即用
- 编排权在 Host，不在业务侧
- 若想手动控制 `role=tool`，扩展空间不自然

## 对外调用形态

### 形态一：provider 创建

```csharp
var provider = ChatProvider.Create(httpClient, settings);
```

感受：

- 统一
- 简单
- 易上手

### 形态二：host 创建 session

```csharp
var host = new AgentHost(provider, options);
var session = host.CreateSession();
```

感受：

- 业务入口看起来是 session
- 实际运行中心是 host

### 形态三：非流式会话调用

```csharp
var result = await session.RunAsync("...");
```

感受：

- 贴近直觉
- 历史消息自动维护

### 形态四：文本流调用

```csharp
await foreach (var chunk in session.StreamTextAsync("..."))
{
    Console.Write(chunk.Text);
}
```

感受：

- 使用自然
- 输出形态简洁

### 形态五：事件流调用

```csharp
await foreach (var evt in session.StreamEventsAsync("..."))
{
}
```

感受：

- 可观测性强
- 语义边界不够直观

## 使用体验拆解

### 入口体验

```text
provider -> host -> session
```

特点：

- 入口对象不止一个
- session 最接近业务句柄
- host 最接近运行编排器

### 历史消息体验

```text
session 内部自动维护 _messages
```

特点：

- 轻量场景顺手
- 高级消息编辑能力不显式

### 工具体验

```text
ToolHub 注册
Host 自动执行
```

特点：

- 上手快
- 自动化程度高
- 业务侧控制权弱

### 流式体验

```text
文本流
事件流
```

特点：

- 文本流对外形态好
- 事件流抽象偏重
