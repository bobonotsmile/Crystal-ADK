# crystal-adk v0.2.0 架构设计稿

## 一、文件树

```text
crystal-adk/
├─ Abstractions/
│  ├─ RuntimeMessage.cs        // 统一消息模型；承载角色、正文、thinking
│  ├─ StreamingChatChunk.cs    // provider 对外的统一流式分片模型；承载正文增量与 thinking 增量
│  └─ TextChunk.cs             // session 对外的最小流分片模型；承载正文文本与 thinking 文本
├─ Providers/
│  ├─ ChatProviderFactory.cs   // provider 工厂入口；按 Vendor 创建具体 provider
│  ├─ ChatProviderOptions.cs   // provider 纯配置数据结构；不负责环境变量或文件读取
│  ├─ Interfaces/
│  │  └─ IChatProvider.cs      // provider 抽象接口；定义非流式与流式两个能力
│  └─ Models/
│     ├─ Ark/
│     │  └─ ArkChatProvider.cs    // ARK provider；负责 HTTP 发送、接收、解析
│     └─ Ollama/
│        └─ OllamaChatProvider.cs // Ollama provider；负责 HTTP 发送、接收、解析
├─ Session/
│  ├─ AgentSession.cs          // 会话操作入口；负责基于消息历史发起非流式/流式对话
│  └─ SessionMessageManager.cs // 消息历史操作入口；负责增删改导出消息
└─ Crystal.Adk.csproj          // 类库项目文件
```

## 三、使用到的设计模式

### Strategy（策略模式：定义一组可替换的做法，对外用同一个抽象接口）
- 代码落点：`Providers/Interfaces/IChatProvider.cs`
- 代码落点：`Providers/Models/Ark/ArkChatProvider.cs`
- 代码落点：`Providers/Models/Ollama/OllamaChatProvider.cs`
- 作用：Session 只依赖 provider 抽象，不依赖具体厂商实现。

### Factory Method（工厂方法：专门有个创建入口，根据条件决定 new 哪个对象）
- 代码落点：`Providers/ChatProviderFactory.cs`
- 作用：根据 `ChatProviderOptions.Vendor` 返回具体 provider。

### Adapter（适配器模式：把一种接口或数据形状，转换成另一种接口或数据形状）
- 代码落点：`Providers/Models/Ark/ArkChatProvider.cs`
- 代码落点：`Providers/Models/Ollama/OllamaChatProvider.cs`
- 作用：将统一消息模型适配为厂商协议，并将厂商响应适配回统一模型。

### Facade（外观模式：对外提供一个更简单的统一入口，把内部复杂性包起来）
- 代码落点：`Session/AgentSession.cs`
- 作用：对外统一暴露会话执行入口，隐藏 provider 调用与历史导出细节。

## 四、总体主链路

```text
开发者手动 new ChatProviderOptions
-> ChatProviderFactory.Create
-> IChatProvider / ArkChatProvider / OllamaChatProvider
-> 开发者手动 new SessionMessageManager
-> AgentSession
-> AgentSession.RunAsync 或 StreamTextAsync
-> ArkChatProvider / OllamaChatProvider
-> 厂商 HTTP API
-> RuntimeMessage / StreamingChatChunk / TextChunk
-> SessionMessageManager 更新消息历史
```

## 五、局部分层

### 抽象层

```text
Abstractions/
  RuntimeMessage
  StreamingChatChunk
  TextChunk
```

定位：

- 统一最小数据结构层
- 不承载厂商协议细节
- 不承载初始化策略

### Provider 层

```text
Providers/
  IChatProvider
  ChatProviderOptions
  ChatProviderFactory
  ArkChatProvider
  OllamaChatProvider
```

定位：

- provider 抽象边界
- provider 配置边界
- provider 工厂入口
- provider 具体发送与解析实现

### Session 层

```text
Session/
  SessionMessageManager
  AgentSession
```

定位：

- `SessionMessageManager`：消息历史编辑与导出
- `AgentSession`：基于历史执行会话

## 六、当前版本结构特征

### 总体特征

- session 是业务侧主操作入口
- 消息历史独立成 `SessionMessageManager`
- provider 配置只保留纯数据字段
- 不内置 `SystemPrompt`
- 不内置 `FromEnvironment()` 一类初始化策略
- `thinking` 输出已经接入统一消息结构与流式分片结构

### 当前局部特征

- `ArkChatProvider.cs` 与 `OllamaChatProvider.cs` 分文件承载厂商协议逻辑
- `adk-lab/Program.cs` 是测试项目装配入口，不是包内核心层
- `thinking` 关闭时不影响原本正文解析链路；只是不会产出 thinking 字段

### 非流式主链路

```text
Program
-> AgentSession.RunAsync
-> SessionMessageManager.AddUser
-> SessionMessageManager.Export
-> IChatProvider.CompleteAsync
-> ArkChatProvider / OllamaChatProvider.BuildRequestBody(stream:false)
-> ArkChatProvider / OllamaChatProvider.CreateHttpRequest
-> HttpClient.SendAsync
-> ArkChatProvider / OllamaChatProvider.ParseAssistantMessage(content + thinking)
-> SessionMessageManager.Add(assistant)
-> RuntimeMessage assistant
```

### 流式主链路

```text
Program
-> AgentSession.StreamTextAsync
-> SessionMessageManager.AddUser
-> SessionMessageManager.Export
-> IChatProvider.StreamAsync
-> ArkChatProvider / OllamaChatProvider.BuildRequestBody(stream:true)
-> ArkChatProvider / OllamaChatProvider.CreateHttpRequest
-> HttpClient.SendAsync(ResponseHeadersRead)
-> ArkChatProvider.ReadStreamAsync / OllamaChatProvider.ReadStreamAsync
-> StreamingChatChunk(TextDelta + ThinkingDelta)
-> TextChunk(Text + ThinkingText)
-> SessionMessageManager.AddAssistant(fullText, fullThinking)
```
